#### 1. Временная сложность (Асимптотика)

Обозначения: $L$ — длина отрезка в пикселях ($\max(|dx|, |dy|)$), $R$ — радиус окружности.

| Алгоритм | Асимптотика | Описание |
| :--- | :--- | :--- |
| **Пошаговый** | $O(L)$ | Выполняется $L$ итераций. Внутри цикла используются операции с плавающей точкой (умножение, сложение) и округление `round()`, что делает константу времени высокой. |
| **ЦДА (DDA)** | $O(L)$ | Выполняется $L$ итераций. Используются операции с плавающей точкой, но только сложение (без умножения внутри цикла), что быстрее пошагового. |
| **Брезенхем (Line)**| $O(L)$ | Выполняется $L$ итераций. Используется **только целочисленная арифметика** (сложение, вычитание, сдвиг). Самый быстрый алгоритм. |
| **Брезенхем (Circle)**| $O(R)$ | Количество шагов пропорционально радиусу (~$R/\sqrt{2}$). Используется только целочисленная арифметика. |
| **Кастла-Питвея** | $O(L)$ | Генерация паттерна занимает время, пропорциональное НОД, отрисовка — пропорциональна длине. Сложная логика генерации делает его медленнее Брезенхема. |
| **Ву (Wu)** | $O(L)$ | Выполняется $L$ итераций. Использует операции с плавающей точкой и отрисовывает по **2 пикселя** на каждый шаг цикла (для сглаживания), что увеличивает время работы. |
